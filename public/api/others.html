Piped

src/Model/Piped.js:19-27

Extends Root

其对 Root 中的 init/update 进行的无差别的赋值， 如果你需要对 View 中的 props 一股脑都同步至 Model 中， 则可以通过继承该类来实现

Root

src/Model/Root.js:20-178

建议所以的 Model 都继承自该类，提供了一些方法

create

src/Model/Root.js:34-36

建议使用给方法创建实例对象，而不是 new Model() 原因参看 https://github.com/imcuttle/babel-plugin-class-properties-default-value

Parameters

  • init object (optional, default {})

Examples

class Model extends Root {
   val = 123
}
const model = Model.create({ val: '345' })

Returns Root

isEmpty

src/Model/Root.js:170-177

  • See: assign

判断当前实例是否内容为空

Examples

Root.create().isEmpty() // true
Root.create({ a: '', b: [] }).isEmpty() // true
Root.create({ a: '', b: [], c: Root.create() }).isEmpty() // true
Root.create({ a: 'a', b: [] }).isEmpty() // false
Root.create({ a: '', b: ['a'] }).isEmpty() // false

Returns Root

Renderer

React.createElement 是不能渲染一个 ViewModel 实例的， 于是便提供了渲染方法来渲染 vm。

h

src/renderer/mixedRenderer.js:38-52

可以渲染 Vm 实例的方法，类似于 React.createElement

Parameters

  • model (VmInstance | ReactComponent | string)
  • propsOrClassName (string | object)
  • props object
  • children ReactElement

Examples

h(model, 'class-name', {}, 
 // children
 '1', '2'
);
h.div({}, 'inner')
h('div', {}, 'inner')
h('div', 'classname', {}, 'inner')
h(Component, 'classname', {}, 
   h.span()
)

Returns ReactElement

src/extension/providerFactory.js:25-35

用于产生一个 Provider，传入一个全局的 store

Examples

const app = AppVM.create()
const Provider = providerFactory(app)

   <Provider>
     {...}
   </Provider>,
   window.root
)
